================================================================================
                           ENDPOINT SUMMARY
================================================================================

AUTHENTICATION (6 endpoints):
  POST   /auth/register               - Register new user
  POST   /auth/login                  - Login and get JWT
  POST   /auth/logout                 - Logout (invalidate token)
  POST   /auth/refresh-token          - Get new JWT token
  POST   /auth/forgot-password        - Request password reset
  POST   /auth/reset-password/:token  - Reset password with token

CLASS MANAGEMENT (4 endpoints):
  POST   /classes                     - Create class (Teacher only)
  GET    /classes                     - List user's classes
  POST   /classes/:classId/join       - Join class (Student only)
  DELETE /classes/:id                 - Delete class (Teacher only)

ATTENDANCE TRACKING (5 endpoints):
  POST   /attendance/start            - Start session (Teacher only)
  PUT    /attendance/end/:sessionId   - End session (Teacher only)
  POST   /attendance/mark             - Mark attendance (Student)
  POST   /attendance/override         - Override record (Teacher only)
  GET    /attendance/reports/:classId - Get reports (Teacher only)

================================================================================
                           AUTHENTICATION
================================================================================

ALL PROTECTED ENDPOINTS REQUIRE:
  Authorization: Bearer <JWT_TOKEN>

OR via Swagger UI:
  1. Click "Authorize" button
  2. Enter JWT token
  3. Click "Authorize"

GETTING A TOKEN:
  1. Register: POST /auth/register
  2. Login: POST /auth/login
  3. Use returned token in Authorization header

TOKEN REFRESH:
  POST /auth/refresh-token with refreshToken
  Get new JWT token when current expires

================================================================================
                          RATE LIMITING
================================================================================

GENERAL API:        100 requests per 15 minutes
LOGIN ATTEMPTS:       5 attempts per 15 minutes
TOKEN REFRESH:       10 attempts per 15 minutes
FORGOT PASSWORD:      5 attempts per 15 minutes

RATE LIMIT HEADERS in response:
  RateLimit-Limit:     Maximum allowed requests
  RateLimit-Remaining: Requests left in window
  RateLimit-Reset:     Unix timestamp when limit resets

                           EXAMPLE USAGE
================================================================================

REGISTER USER:
  curl -X POST http://localhost:5000/api/v1/auth/register \
    -H "Content-Type: application/json" \
    -d '{
      "name": "John Doe",
      "email": "john@example.com",
      "password": "SecurePass123!",
      "role": "Student"
    }'

LOGIN:
  curl -X POST http://localhost:5000/api/v1/auth/login \
    -H "Content-Type: application/json" \
    -d '{
      "email": "john@example.com",
      "password": "SecurePass123!"
    }'

GET CLASSES (with JWT):
  TOKEN="eyJhbGciOiJIUzI1NiIs..."
  curl -X GET http://localhost:5000/api/v1/classes \
    -H "Authorization: Bearer $TOKEN"

CREATE CLASS (Teacher only):
  curl -X POST http://localhost:5000/api/v1/classes \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d '{
      "name": "Physics 101",
      "description": "Introduction to Physics"
    }'

START ATTENDANCE:
  curl -X POST http://localhost:5000/api/v1/attendance/start \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d '{
      "classId": "60d5ec49c1234567890abcde"
    }'
